# DocMind Skill

## 需求分析

### 项目目标
自动分析项目结构，为每个文件夹和重要文件生成面向小白的说明文档。

### 核心功能
1. **项目扫描**：递归遍历所有目录，识别文件类型
2. **文档生成**：为每个文件夹生成 `说明.md`
3. **代码注释**：为代码文件添加头部注释
4. **总览输出**：在根目录生成 `项目总览.md` 和 `文件关系图.md`

### 目标用户
- 完全不懂代码的小白
- 需要快速了解项目的非技术人员
- 新加入团队的成员

### 用户场景
1. **小白查看项目**：打开项目总览.md，快速了解项目概况
2. **探索具体模块**：进入某个文件夹，查看说明.md了解详情
3. **阅读代码**：通过文件头部注释理解代码用途

### 非功能需求
- **安全**：不修改代码逻辑，只添加注释
- **可配置**：支持排除特定文件/文件夹
- **可扩展**：支持多种编程语言的注释格式
- **小白友好**：避免技术术语，用通俗语言

## 技术设计

### 模块划分

```
src/
├── main.ts              # 入口文件，协调各模块
├── scanner/             # 扫描模块
│   └── project-scanner.ts
├── analyzer/            # 分析模块
│   └── file-analyzer.ts
├── generator/           # 生成模块
│   └── doc-generator.ts
├── commenter/           # 注释模块
│   └── code-commenter.ts
├── prompts/             # 提示词模板
│   ├── templates.ts
│   ├── folder-doc-prompt.md
│   ├── overview-prompt.md
│   └── code-comment-prompt.md
└── utils/               # 工具函数
    ├── config-loader.ts
    ├── file-types.ts
    └── logger.ts
```

### 核心流程

```
用户执行 /docmind
    ↓
[ConfigLoader] 加载配置
    ↓
[ProjectScanner] 扫描项目结构
    ↓
[FileAnalyzer] 分析文件内容和关系
    ↓
[DocGenerator] 生成说明文档
    ↓
[CodeCommenter] 添加代码注释
    ↓
完成，输出统计信息
```

### 数据结构

```typescript
// 项目结构
interface ProjectStructure {
  rootPath: string;
  folders: FolderInfo[];
  files: FileInfo[];
}

// 分析后的项目
interface AnalyzedProject {
  rootPath: string;
  folders: AnalyzedFolder[];
  files: AnalyzedFile[];
  relationships: FileRelationship[];
}
```

## 输出规范

### 文件夹说明.md
- 文件夹用途（通俗语言）
- 文件清单及说明
- 子文件夹列表
- 重要提示

### 项目总览.md
- 项目简介（2-3句话）
- 核心目录列表
- 主要文件列表
- 统计信息
- 阅读指引

### 代码注释
- 文件用途说明
- 主要功能列表
- 使用场景说明

## 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 文件过多导致性能问题 | 高 | 限制扫描深度，支持分批处理 |
| 已有注释被覆盖 | 高 | 先检查是否已有注释 |
| 生成的描述不准确 | 中 | 基于文件名和结构推断，允许手动修正 |
| 多语言项目支持不全 | 低 | 逐步扩展支持的编程语言 |
