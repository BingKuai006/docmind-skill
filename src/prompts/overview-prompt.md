# DocMind 项目总览模板

## 用途说明

在根目录生成 `项目总览.md`，给小白一个完整的项目概览。

## 模板格式

```markdown
# 项目总览

## 这个项目是干什么的？

[2-3句话概括项目的主要功能和用途，用最通俗的语言]

## 项目结构概览

### 核心目录

- **[目录名]/** - [用一句话说明这个目录的作用]
- **[目录名]/** - [用一句话说明这个目录的作用]

### 主要文件

- **[文件名]** - [这个文件的作用]
- **[文件名]** - [这个文件的作用]

## 代码统计

- 总文件夹数：[数量]
- 总文件数：[数量]
- 代码文件数：[数量]
- 配置文件数：[数量]
- 文档文件数：[数量]

## 如何开始？

1. 先查看 [文件关系图.md] 了解文件之间的联系
2. 然后查看各文件夹中的 [说明.md] 了解具体内容
3. [针对具体项目的阅读建议]

---
*由 DocMind 自动生成*
```

## 写作原则

1. **首段最重要**：2-3句话说清楚项目是干什么的
2. **突出核心**：只列出最重要的目录和文件
3. **数字说话**：用统计数据增强直观感受
4. **指引清晰**：告诉用户下一步该看什么

## 示例

```markdown
# 项目总览

## 这个项目是干什么的？

这是一个电商网站的后台管理系统。商家可以用它来管理商品、处理订单、查看销售数据。普通用户可以通过前台页面浏览商品并下单购买。

## 项目结构概览

### 核心目录

- **src/** - 存放所有程序代码
- **api/** - 处理数据请求的接口
- **database/** - 存放数据存储相关的代码

### 主要文件

- **main.ts** - 程序入口，启动整个应用
- **config.json** - 存储应用配置信息

## 代码统计

- 总文件夹数：12
- 总文件数：156
- 代码文件数：89
- 配置文件数：23
- 文档文件数：44

## 如何开始？

1. 先查看 [文件关系图.md] 了解文件之间的联系
2. 然后查看各文件夹中的 [说明.md] 了解具体内容
3. 从 src/main.ts 开始阅读，了解程序如何启动

---
*由 DocMind 自动生成*
```
