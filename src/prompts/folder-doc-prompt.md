# DocMind 文件夹说明模板

## 用途说明

为每个文件夹生成一个 `说明.md` 文件，帮助小白理解项目结构。

## 模板格式

```markdown
# [文件夹名称]

## 这个文件夹是干什么的？

[用通俗的语言解释这个文件夹的用途，避免技术术语]

## 里面有什么？

### 代码文件 ([数量]个)

- **[文件名]** - [这个文件是干什么的]

### 配置文件 ([数量]个)

- **[文件名]** - [配置文件的作用]

### 其他文件 ([数量]个)

- **[文件名]** - [文件类型和用途]

## 子文件夹

- [子文件夹1]/
- [子文件夹2]/

## 重要说明

- ⚠️ **核心目录**：这个文件夹包含重要的代码文件
- ℹ️ **辅助目录**：提供辅助功能支持
- 📌 **主要文件**：[文件名1], [文件名2]

---
*由 DocMind 自动生成*
```

## 写作原则

1. **小白友好**：不用 "依赖注入"、"中间件" 等专业术语
2. **具体描述**：不说 "处理逻辑"，而说 "保存用户数据"
3. **举例说明**：提供实际使用场景
4. **避免重复**：不要每个文件都说 "这是一个...文件"

## 示例

### 好的描述

```
这个文件夹存放所有页面的组件。
- Button.tsx - 按钮组件，用户点击后触发操作
- Form.tsx - 表单组件，收集用户输入的信息
```

### 不好的描述

```
这个文件夹包含UI组件。
- Button.tsx - 按钮组件
- Form.tsx - 表单组件
```
