# DocMind 代码注释模板

## 用途说明

为代码文件添加头部注释，用小白能理解的语言说明文件用途。

## JavaScript/TypeScript 注释格式

```typescript
/**
 * [文件名]
 * 
 * [这个文件的主要功能，1-2句话]
 * 
 * 主要功能：
 * - [功能1的通俗描述]
 * - [功能2的通俗描述]
 * - [功能3的通俗描述]
 * 
 * 什么时候会用到：
 * - [场景1：什么时候会用到这个文件]
 * - [场景2：什么时候会用到这个文件]
 */
```

## Python 注释格式

```python
"""[文件名]

[这个文件的主要功能，1-2句话]

主要功能：
- [功能1的通俗描述]
- [功能2的通俗描述]
- [功能3的通俗描述]

什么时候会用到：
- [场景1：什么时候会用到这个文件]
- [场景2：什么时候会用到这个文件]
"""
```

## Java/C# 注释格式

```java
/**
 * [文件名]
 * 
 * [这个文件的主要功能，1-2句话]
 * 
 * 主要方法：
 * - [方法1的通俗描述]
 * - [方法2的通俗描述]
 * - [方法3的通俗描述]
 * 
 * @author DocMind
 */
```

## 写作原则

1. **避免术语**：不用 "实例化"、"回调函数"、"异步处理"
2. **具体场景**：说明 "保存用户填写的表单数据" 而不是 "处理输入"
3. **最多5个功能**：不要罗列所有函数
4. **真实场景**：什么时候会用到要具体，比如 "用户点击登录按钮时"

## 示例

### 好的注释

```typescript
/**
 * auth.ts
 * 
 * 这个文件负责用户的登录和注册功能。
 * 
 * 主要功能：
 * - 检查用户输入的账号密码是否正确
 * - 为新用户创建账号
 * - 生成登录凭证，让用户保持登录状态
 * 
 * 什么时候会用到：
 * - 用户在登录页面输入账号密码时
 * - 用户点击"注册"按钮创建新账号时
 */
```

### 不好的注释

```typescript
/**
 * auth.ts
 * 
 * 用户认证模块。
 * 
 * 主要功能：
 * - login()
 * - register()
 * - validateToken()
 * - hashPassword()
 * - generateJWT()
 * 
 * 什么时候会用到：
 * - 处理认证请求
 * - 验证用户身份
 */
```

## 注意事项

- 如果文件已经有注释，不要覆盖
- 不要修改代码逻辑，只添加注释
- 多语言项目根据文件扩展名选择对应格式
